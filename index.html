  • Conditional Rendering

    Showing and hiding (or creating / destroying) elements based on custom rules

    • v-if

      Showing and hiding element by v-if directive with alternative example in mustaches

    • Template v-if

      Template is aspecial - tag (wrapper) which allow you to apply one directive to many elements

    • v-show

      Elements with v-show are Always rendered to DOM and just toggle display property

    • v-else

      v-else is a helper directive for if and show, activated on next element when previous states are not matched

      • Component caveat

        Using "condition" and "!condition" as an alternative to else, good when mixing rules on different element types

    • v-if vs. v-show

      v-show is faster when you toggle element often, v-if doesn't mess html and destroys child elements completely

  • List Rendering

    Rendering list of items on a website from JS Array or Object

    • v-for

      Directive for looping through data to render some listed elements

    • Template v-for

      Usign template tag to avoid messing rendered html with directives

    • Array Change Detection

      Vue watches for changes in rendered array and refresh rendered element every time the array is changed

      • Mutation Methods

        List of array mutation methods wrapped by Vue to watch for changes

      • Replacing an Array

        Replacing array with another array (for i.e. filtered from some data) is very effective in Vue

      • track-by

        Faster way of re-rendering objects in array by telling to render only the new ones by uniqueId

      • track-by $index

        Another way to re-render only new items, very fast but may lose DOM input values and component states

      • Caveats

        Examples when vue can't track changes in Array, like modifying item by index... items[12]

    • Object v-for

      Vue can also render from object, not only from arrays, that's cool

    • Range v-for

      Looping just a number of times

    • Displaying Filtered/Sorted Results

      Two ways to filter some results from entire Array. Easier - by filters and advanced by creating new property

      • Checkbox

    • Param Attributes

    • What are Components?

      description

    • Using Components

      • Registration

        Registering new "my-component" tag which is then rendered as pure html code. Three steps: 1. define template, 2. register custom tag, 3. create a root for components.

      • Local Registration

        Registering new components which will work only inside another component (Registering them locally).

      • Registration Sugar

        Shorter code for registering components: defining template and registering tagName at once

      • Component Option Caveats

        Avoiding sharing block of data between multiple instances component, each instance should have it's own data.

      • Template Parsing

        Additional information where you can't use your custom tags, for example you can't put clickable button inside a href element.

    • Props

      Component properties can contain data for custom tags. This data can be passed from Html or inherited from parent components.

      • Passing Data with Props

        Custom tag properties which look like custom html attributes in custom tag (my-component color="red") should be defined along with component template.

      • camelCase vs. kebab-case

        Even if you name property "likeThis" in JS, it will be available as "like-this" attribute in html.

      • Dynamic Props

        Binding component Html value to it's JS properties, using v-bind and rendering property using v-model.

      • Literal vs. Dynamic

        custom-property passes value to JS as a string and :custom-roperty passes it as a real JS number.

      • Prop Binding Types

        Properties are passed from parent components to childs, but you can control ways of passing props with :attr, :attr.sync and :attr.once

      • Prop Validation

        You can ensure properties are passed to component in as a proper JS type, such as String or Bool and automatically transform the passed value.

    • Parent-Child Communication

      description

      • Parent Chain

        Each component has access to it's root, parent and children (this.$parent, this.$root, this.$children)

      • Custom Events

        Creating events for component ($on), triggering ($emit), sending to parents ($dispatch), sending downwards ($broadcast).

      • v-on for Custom Events

        Calling a method in parent element with custom v-on event

      • Child Component Refs

        Getting (mirroring) data from child component by setting reference v-ref.
